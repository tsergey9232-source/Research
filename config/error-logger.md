# 🔴 Системный логгер — технические логи pipeline

## Назначение

Архивариус ведёт чистые читаемые протоколы для Сергея.
Системный логгер ведёт технические логи для Crabe — отладка, анализ качества, улучшение алгоритмов.

---

## Файловая структура логов

Каждый прогон: `results/{run}/logs/`

```
logs/
├── pipeline-log.md          ← чистый лог для Сергея (этапы, статусы)
├── dispatcher-log.md        ← технический лог оркестратора (таймауты, перезапуски)
├── error-log.md             ← ВСЕ ошибки и проблемы (этот файл)
├── quality-log.md           ← качество данных по каждому агенту
└── agent-{id}.log           ← лог каждого конкретного агента
```

---

## Формат error-log.md

```
[HH:MM:SS] [УРОВЕНЬ] [АГЕНТ] [ЭТАП] ОПИСАНИЕ

Уровни:
  ERROR   — критическая ошибка, агент не выполнил задачу
  WARN    — предупреждение, агент выполнил частично
  TIMEOUT — агент завис и был перезапущен
  RETRY   — агент перезапущен (попытка N)
  SKIP    — агент пропущен (причина)
  QUALITY — оценка качества данных от чистильщика
```

Пример:
```
[14:05:33] [TIMEOUT] [researcher-07] [research/idea-03] Агент завис на 45 мин, перезапущен
[14:07:11] [ERROR]   [researcher-07] [research/idea-03] Повторный сбой после перезапуска, пропущен
[14:12:44] [QUALITY] [researcher-02] [research/idea-03] Мусор: 18% (4 битых ссылки из 22)
[14:13:01] [WARN]    [analyst-04]    [analysis/idea-03] Заключение слишком короткое (<200 слов), отмечено
```

## Формат quality-log.md

Ведётся Чистильщиком после обработки каждого исследователя.

```
Идея #{N} — {название}
────────────────────────────────
Агент               | Ссылок | Удалено | Мусор% | Качество
researcher-01       | 34     | 5       | 14.7%  | ★★★★☆
researcher-02       | 18     | 2       | 11.1%  | ★★★★★
researcher-03       | 9      | 4       | 44.4%  | ★★☆☆☆  ← низкое
...
Итого               | 245    | 31      | 12.7%  | ★★★★☆
Проблемные агенты:  researcher-03 (44% мусора — требует ревью алгоритма)
```

---

## Что логируется обязательно

### По каждому агенту:
- Время запуска и завершения
- Количество попыток (если были перезапуски)
- Код завершения (OK / TIMEOUT / ERROR / SKIP)
- Объём выходных данных (строк, байт)

### По исследователям дополнительно:
- Количество найденных ссылок
- Процент удалённых (мусор)
- Процент битых ссылок
- Языки, на которых искал

### По аналитикам дополнительно:
- Объём заключения (кол-во слов)
- Наличие ссылок на источники (да/нет)
- Субъективная оценка Верховного аналитика (1-5)

### По жюри дополнительно:
- Наличие комментариев к оценкам (да/нет)
- Разброс оценок (если один оценил 9, другой 2 — аномалия)

---

## Агент-логгер (роль)

**Кто:** Crabe (я) — веду все технические логи сам в процессе оркестрации
**Когда:** в реальном времени, сразу при событии
**Формат:** строго по шаблону выше
**Хранение:** в `logs/error-log.md` и `logs/quality-log.md` текущего прогона

---

## Анализ после прогона

После завершения pipeline Crabe делает краткий технический разбор:
- Какие агенты дали больше всего мусора
- Кто завис и почему
- Какие источники оказались наименее полезными
- Рекомендации по улучшению на следующий прогон

Сохраняется в: `logs/post-run-analysis.md`
