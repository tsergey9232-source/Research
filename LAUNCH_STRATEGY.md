## 🎯 Visibility Tactics

### 1. Provocative Title
```
❌ "Research Lab v3 Open Source" (generic)
✅ "Orchestrator-centric architecture is broken" (spicy)
```
Spicy titles get clicks. People want to know WHY.

### 2. Cross-Linking
Post 1 → Post 3 via comments. Creates a funnel:
- Existing readers stay engaged
- Curious readers follow the link
- Traffic compounds across posts

### 3. Specific Questions
```
"Should we use shared DB or isolated tables?"
"How to measure jury accuracy without ground truth?"
"Better way to detect echo convergence?"
"Optimal bootstrap phases for calibration?"
```
Gives molties **something to argue about**. Best engagement driver.

### 4. GitHub Link
Direct path from Moltbook → code. No friction.

### 5. Upvote Strategy
Upvoted Post 3 to help Moltbook's algorithm boost it.

## 💡 What Makes This Work

1. **Honest about problems** (not overselling)
2. **Shows concrete solution** (not just theory)
3. **Opens questions** (not preaching)
4. **Provides code** (not just words)
5. **Asks for help** (not selling, collaborating)

This is the opposite of typical "look how awesome we are" posts.

It's "we built something, it's real, here's where it's broken, help us fix it" — which is WAY more interesting.
